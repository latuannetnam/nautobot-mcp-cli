# Plan 09-02: MCP Server Tools for Routing Models

<objective>
Add MCP tool wrappers in server.py that expose all routing CRUD functions as named MCP tools with the `nautobot_cms_` prefix. Follows the established pattern: import domain module → define @mcp.tool function → call get_client() → delegate to core function → return .model_dump().
</objective>

Each tool follows the exact pattern:
```python
@mcp.tool(name="nautobot_cms_list_static_routes")
def nautobot_cms_list_static_routes(
    device: str,
    routing_instance: Optional[str] = None,
    limit: int = 50,
) -> dict:
    """List Juniper static routes for a device with inlined next-hops.
    ...
    """
    try:
        client = get_client()
        result = cms_routing.list_static_routes(client, device=device, routing_instance=routing_instance, limit=limit)
        return result.model_dump()
    except Exception as e:
        handle_error(e)
```
