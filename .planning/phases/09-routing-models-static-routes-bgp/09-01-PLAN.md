Create `nautobot_mcp/models/cms/routing.py` with Pydantic models:

1. **StaticRouteSummary** (extends CMSBaseSummary):
   - Fields: `destination` (str), `routing_table` (str, default "inet.0"), `address_family` (str), `preference` (int), `metric` (int), `enabled` (bool), `discarded` (bool), `rejected` (bool), `communities` (str), `routing_instance_name` (Optional[str]), `routing_instance_id` (Optional[str]), `is_active` (Optional[bool]), `route_state` (Optional[str])
   - Additional fields for inlined next-hops: `nexthops` (list[StaticRouteNexthopSummary], default []), `qualified_nexthops` (list[StaticRouteQualifiedNexthopSummary], default [])
   - `from_nautobot(record)` extracts all fields including routing_instance nested object

2. **StaticRouteNexthopSummary** (extends CMSBaseSummary):
   - Fields: `route_id` (str), `ip_address` (str), `is_active_nexthop` (Optional[bool]), `weight` (int, default 1), `lsp_name` (str, default ""), `mpls_label` (str, default ""), `nexthop_type` (str, default ""), `via_interface_name` (Optional[str])
   - `from_nautobot(record)` extracts nested ip_address and via_interface display values

3. **StaticRouteQualifiedNexthopSummary** (extends CMSBaseSummary):
   - Same fields as StaticRouteNexthopSummary plus: `interface_name` (Optional[str])
   - `from_nautobot(record)` extracts nested interface and ip_address

4. **BGPGroupSummary** (extends CMSBaseSummary):
   - Fields: `name` (str), `description` (str, default ""), `type` (str), `local_address` (Optional[str]), `cluster_id` (str, default ""), `authentication_algorithm` (str, default ""), `routing_instance_name` (Optional[str]), `routing_instance_id` (Optional[str]), `enabled` (bool), `neighbor_count` (Optional[int])

5. **BGPNeighborSummary** (extends CMSBaseSummary):
   - Fields: `group_id` (str), `group_name` (Optional[str]), `peer_ip` (Optional[str]), `description` (str, default ""), `peer_as` (Optional[int]), `local_address` (Optional[str]), `remove_private_as` (bool, default False), `as_override` (bool, default False), `enabled` (bool), `session_state` (str, default ""), `received_prefix_count` (Optional[int]), `sent_prefix_count` (Optional[int]), `flap_count` (Optional[int])

6. **BGPAddressFamilySummary** (extends CMSBaseSummary):
   - Fields: `group_id` (Optional[str]), `neighbor_id` (Optional[str]), `address_family` (str), `sub_address_family` (str), `enabled` (bool), `prefix_limit_max` (Optional[int]), `prefix_limit_teardown` (bool, default False)

7. **BGPPolicyAssociationSummary** (extends CMSBaseSummary):
   - Fields: `bgp_group_id` (Optional[str]), `bgp_neighbor_id` (Optional[str]), `policy_id` (str), `policy_name` (Optional[str]), `policy_type` (str), `order` (int)

8. **BGPReceivedRouteSummary** (extends CMSBaseSummary):
   - Fields: `neighbor_id` (str), `routing_table` (str), `prefix` (str), `is_active` (bool, default False), `as_path` (str, default ""), `local_preference` (Optional[int]), `med` (Optional[int]), `next_hop` (Optional[str]), `origin` (str, default ""), `communities` (str, default "")

Each model's `from_nautobot()` classmethod must extract nested FK objects (device, routing_instance, ip_address, prefix, group, policy) using `getattr` with fallback to dict access, following the pattern in `CMSBaseSummary._extract_device()`.

### Task 3: Create routing CRUD functions

<read_first>
- nautobot_mcp/cms/client.py (cms_list, cms_get, cms_create, cms_update, cms_delete helpers, resolve_device_id)
- nautobot_mcp/models/cms/routing.py (the models just created)
- nautobot_mcp/devices.py (existing list/get/create/update/delete pattern)
</read_first>

<action>
Create `nautobot_mcp/cms/routing.py` with these functions:

**Static Routes (full CRUD):**
- `list_static_routes(client, device, routing_instance=None, limit=0)` → `ListResponse[StaticRouteSummary]`
  - Resolves device name to UUID via `resolve_device_id`
  - Calls `cms_list(client, "juniper_static_routes", StaticRouteSummary, device=device_id, ...)`
  - Then batch-fetches all nexthops and qualified nexthops for the device
  - Attaches nexthops to each route by matching `route` FK UUID
  - Returns routes with inlined nexthops
- `get_static_route(client, id)` → `StaticRouteSummary` (with inlined nexthops)
- `create_static_route(client, device, destination, routing_table="inet.0", preference=5, **kwargs)` → `StaticRouteSummary`
- `update_static_route(client, id, **updates)` → `StaticRouteSummary`
- `delete_static_route(client, id)` → `dict`

**Static Route Nexthops (list/get only):**
- `list_static_route_nexthops(client, route_id=None, device=None, limit=0)` → `ListResponse[StaticRouteNexthopSummary]`
- `get_static_route_nexthop(client, id)` → `StaticRouteNexthopSummary`
- `list_static_route_qualified_nexthops(client, route_id=None, device=None, limit=0)` → `ListResponse[StaticRouteQualifiedNexthopSummary]`
- `get_static_route_qualified_nexthop(client, id)` → `StaticRouteQualifiedNexthopSummary`

**BGP Groups (full CRUD):**
- `list_bgp_groups(client, device, routing_instance=None, limit=0)` → `ListResponse[BGPGroupSummary]`
  - Device-scoped: resolves device name, filters by device UUID
- `get_bgp_group(client, id)` → `BGPGroupSummary`
- `create_bgp_group(client, device, name, type, **kwargs)` → `BGPGroupSummary`
- `update_bgp_group(client, id, **updates)` → `BGPGroupSummary`
- `delete_bgp_group(client, id)` → `dict`

**BGP Neighbors (full CRUD):**
- `list_bgp_neighbors(client, device=None, group_id=None, limit=0)` → `ListResponse[BGPNeighborSummary]`
  - When device is given: fetch all groups for device, then filter neighbors by those group IDs
  - When group_id is given: filter directly by group
- `get_bgp_neighbor(client, id)` → `BGPNeighborSummary`
- `create_bgp_neighbor(client, group_id, peer_ip, peer_as=None, **kwargs)` → `BGPNeighborSummary`
- `update_bgp_neighbor(client, id, **updates)` → `BGPNeighborSummary`
- `delete_bgp_neighbor(client, id)` → `dict`

**BGP Address Families (list/get only):**
- `list_bgp_address_families(client, group_id=None, neighbor_id=None, limit=0)` → `ListResponse[BGPAddressFamilySummary]`
- `get_bgp_address_family(client, id)` → `BGPAddressFamilySummary`

**BGP Policy Associations (list/get only):**
- `list_bgp_policy_associations(client, group_id=None, neighbor_id=None, limit=0)` → `ListResponse[BGPPolicyAssociationSummary]`
- `get_bgp_policy_association(client, id)` → `BGPPolicyAssociationSummary`

**BGP Received Routes (list/get only):**
- `list_bgp_received_routes(client, neighbor_id=None, limit=0)` → `ListResponse[BGPReceivedRouteSummary]`
- `get_bgp_received_route(client, id)` → `BGPReceivedRouteSummary`

All functions follow the pattern:
```python
def list_X(client: NautobotClient, ...) -> ListResponse[XSummary]:
    try:
        # resolve device if needed
        # call cms_list with appropriate endpoint and model
        return result
    except Exception as e:
        client._handle_api_error(e, "list", "ModelName")
        raise
```
</action>
