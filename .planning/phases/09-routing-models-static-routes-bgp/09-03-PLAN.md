2. Register it in `nautobot_mcp/cli/app.py`:
   - Create a `cms_app = typer.Typer(name="cms", help="CMS plugin operations")` if not existing
   - Create a `routing_app = typer.Typer(name="routing", help="Juniper routing model operations")`
   - Add `cms_app.add_typer(routing_app)`
   - Add `app.add_typer(cms_app)`

3. CLI commands in `cms_routing.py` (all accept `--device` required param):

**Static Routes:**
- `list-static-routes --device NAME [--routing-instance NAME] [--limit 50] [--detail]`
  - Default table columns: Destination | Routing Table | Preference | Action | NH Count | Active
  - `--detail` adds: Next-hop IPs, Qualified next-hop IPs, Communities
- `get-static-route --id UUID [--detail]`
- `create-static-route --device NAME --destination PREFIX [--routing-table inet.0] [--preference 5]`
- `delete-static-route --id UUID [--confirm/--yes]`

**BGP Groups:**
- `list-bgp-groups --device NAME [--routing-instance NAME] [--limit 50]`
  - Table columns: Name | Type | Local Address | Cluster ID | Neighbors | Enabled
- `get-bgp-group --id UUID`
- `create-bgp-group --device NAME --name TEXT --type internal|external [--local-address IP] [--cluster-id TEXT]`
- `delete-bgp-group --id UUID [--confirm/--yes]`

**BGP Neighbors:**
- `list-bgp-neighbors --device NAME [--group-id UUID] [--limit 50]`
  - Table columns: Peer IP | Peer AS | Group | State | Rcvd Pfx | Sent Pfx | Enabled
- `get-bgp-neighbor --id UUID`
- `create-bgp-neighbor --group-id UUID --peer-ip IP [--peer-as INT] [--description TEXT]`
- `delete-bgp-neighbor --id UUID [--confirm/--yes]`

**Read-only child models:**
- `list-bgp-address-families [--group-id UUID] [--neighbor-id UUID] [--limit 50]`
  - Table columns: AFI | SAFI | Parent | Enabled | Prefix Limit
- `list-bgp-policy-associations [--group-id UUID] [--neighbor-id UUID] [--limit 50]`
  - Table columns: Policy | Type | Order | Parent
- `list-bgp-received-routes --neighbor-id UUID [--limit 50]`
  - Table columns: Prefix | Active | AS Path | LP | MED | Next Hop

Each command uses `rich.table.Table` for tabular output and `rich.console.Console` for `--detail` output.
All commands use the pattern:
```python
@routing_app.command("list-static-routes")
def list_static_routes_cmd(device: str = typer.Option(..., help="Device name"), ...):
    """List Juniper static routes for a device."""
    client = get_client_from_config()
    result = cms_routing.list_static_routes(client, device=device, ...)
    # Format as rich table
```
</action>
