# Plan 19-02: Composite Function Partial Failure Refactoring

<objective>
Refactor all 4 composite workflow functions (`get_device_bgp_summary`, `get_device_routing_table`, `get_device_firewall_summary`, `get_interface_detail`) to use `WarningCollector` for graceful degradation. </objective>

- All 4 composite functions return `(result, warnings)` tuples
- All existing silent `except Exception: pass` blocks upgraded to capture warnings via collector
- Primary query failures still raise exceptions (caught by `run_workflow` → `error`)
- Enrichment failures produce warnings and continue execution
- `firewall_summary` treats filters and policers as independent co-primaries
- `interface_detail` per-unit enrichment failures produce per-unit warnings
- All existing tests pass; new tests validate partial failure for each composite
</must_haves>

<action>
Modify `get_device_firewall_summary()` in `nautobot_mcp/cms/firewalls.py`:

1. Add import at top of file:
```python
from nautobot_mcp.warnings import WarningCollector
```

2. Create `WarningCollector` at start of function.

3. **Implement independent co-primaries pattern.** Currently the outer try/except wraps BOTH `list_firewall_filters` and `list_firewall_policers`. Refactor to try each independently:
```python
    collector = WarningCollector()
    filters_data = []
    policers_data = []
    filters_ok = False
    policers_ok = False

    # Co-primary 1: Filters
    try:
        filters_data = list_firewall_filters(client, device=device)
        filters_ok = True
    except Exception as e:
        collector.add("list_firewall_filters", str(e))

    # Co-primary 2: Policers
    try:
        policers_data = list_firewall_policers(client, device=device)
        policers_ok = True
    except Exception as e:
        collector.add("list_firewall_policers", str(e))

    # If BOTH co-primaries fail → raise to produce error envelope
    if not filters_ok and not policers_ok:
        raise RuntimeError("Both primary queries failed: filters and policers")
```

4. Upgrade enrichment silent try/except blocks (term inlining for filters, policer action inlining) to capture warnings.

5. Return tuple:
```python
    return result, collector.warnings
```
</action>

- If only filters fail → result has policers + warning, status becomes `partial`

- If BOTH fail → exception raised → `error` status
