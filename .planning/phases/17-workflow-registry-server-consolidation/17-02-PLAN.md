5. **Tool 1: `nautobot_api_catalog`**
   ```python
   @mcp.tool(name="nautobot_api_catalog")
   def nautobot_api_catalog(domain: Optional[str] = None) -> dict:
       """Discover available Nautobot API endpoints and workflows.

       Returns endpoints grouped by domain with methods, filters, and
       descriptions. Use this to find the right endpoint before calling
       call_nautobot or run_workflow.

       Args:
           domain: Optional filter — dcim, ipam, circuits, tenancy, cms, workflows.

       Returns:
           Catalog dict grouped by domain.
       """

           return get_catalog(domain=domain)
       except Exception as e:
           handle_error(e)
   ```

6. **Tool 2: `nautobot_call_nautobot`**
   ```python
   @mcp.tool(name="nautobot_call_nautobot")
   def nautobot_call_nautobot(
       endpoint: str,
       method: str = "GET",
       params: Optional[dict] = None,
       data: Optional[dict] = None,
       id: Optional[str] = None,
       limit: int = 50,
   ) -> dict:
       """Execute any Nautobot API operation — universal CRUD dispatcher.

       Routes to core Nautobot (/api/*) or CMS plugin (cms:*) backends.
       Validates endpoint against catalog with 'did you mean?' hints.

       Args:
           endpoint: API path ("/api/dcim/devices/") or CMS key ("cms:juniper_bgp_groups").
           method: HTTP method — GET, POST, PATCH, DELETE.
           params: Query filters for GET.
           data: Request body for POST/PATCH.
           id: Object UUID for single-object ops.
           limit: Max GET results (default 50, cap 200).

       Returns:
           Response dict with count, results, endpoint, method.
       """
       try:
           client = get_client()
           return call_nautobot(client, endpoint=endpoint, method=method,
                                params=params, data=data, id=id, limit=limit)
       except Exception as e:
           handle_error(e)
   ```

7. **Tool 3: `nautobot_run_workflow`**
   ```python
   @mcp.tool(name="nautobot_run_workflow")
   def nautobot_run_workflow(
       workflow: str,
       params: Optional[dict] = None,
   ) -> dict:
       """Execute a named server-side workflow.

       Workflows are composite operations that aggregate multiple API calls
       into a single response (e.g., BGP summary, device drift comparison).
       Use nautobot_api_catalog(domain='workflows') to see available workflows.

       Args:
           workflow: Workflow name (e.g., "bgp_summary", "compare_device").
           params: Workflow-specific parameters dict.

       Returns:
           Envelope dict with workflow, device, status, data, timestamp.
       """
       try:
           client = get_client()
           return run_workflow(client, workflow_id=workflow, params=params or {})
       except Exception as e:
           handle_error(e)
   ```

The file should be ~120-150 lines total including module docstring, blank lines, and comments.
