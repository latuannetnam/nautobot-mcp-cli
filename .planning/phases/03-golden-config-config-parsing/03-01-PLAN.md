Add a `golden_config` property to `NautobotClient` that accesses the Golden Config plugin endpoints via pynautobot.

Add this property to the NautobotClient class:
```python
@property
def golden_config(self):
    """Access Golden Config plugin endpoints.
    
    Returns pynautobot plugin app for golden-config.
    Raises NautobotAPIError if plugin is not installed.
    """
    try:
        return self.api.plugins.golden_config
    except Exception as e:
        raise NautobotAPIError(
            message="Golden Config plugin not available",
            code="PLUGIN_NOT_FOUND",
            hint="Ensure nautobot-golden-config is installed on your Nautobot instance"
        )
```

This follows the same lazy property pattern as existing `dcim`, `ipam`, `tenancy`, and `circuits` properties.

class ComplianceFeatureSummary(BaseModel):
    id: str
    name: str
    slug: str
    description: str = ""

    @classmethod
    def from_nautobot(cls, nb_record) -> "ComplianceFeatureSummary":
        return cls(
            id=str(nb_record.id),
            name=nb_record.name,
            slug=nb_record.slug,
            description=getattr(nb_record, 'description', '') or '',
        )
```

class ComplianceFeatureResult(BaseModel):

    missing_lines: str = ""
    extra_lines: str = ""
    actual: str = ""
    intended: str = ""
```

5. **ComplianceResult** — Full compliance report for a device:
```python
class ComplianceResult(BaseModel):
    device: str
    overall_status: str  # "compliant", "non-compliant", "pending"
    features: list[ComplianceFeatureResult] = []
    checked_at: str = ""
    source: str = ""  # "server" or "quick-diff"

    @classmethod
    def from_nautobot(cls, device_name: str, nb_records) -> "ComplianceResult":
        features = []
        for r in nb_records:
            features.append(ComplianceFeatureResult(
                feature=str(r.feature),
                status="compliant" if r.compliance else "non-compliant",
                ordered=getattr(r, 'ordered', False),
                missing_lines=getattr(r, 'missing', '') or '',
                extra_lines=getattr(r, 'extra', '') or '',
                actual=getattr(r, 'actual', '') or '',
                intended=getattr(r, 'intended', '') or '',
            ))
        overall = "compliant" if all(f.status == "compliant" for f in features) else "non-compliant"
        return cls(device=device_name, overall_status=overall, features=features, source="server")
```

Create `nautobot_mcp/golden_config.py` with the following functions. All follow the established pattern: `client: NautobotClient` as first param, return pydantic models.

**Config Retrieval (GC-01, GC-02):**

1. `get_intended_config(client, device_name_or_id: str) -> GoldenConfigEntry`
   - Filter golden_config by device, return intended_config field
   - `configs = client.golden_config.golden_config.filter(device=device_name_or_id)`
   - Return first result wrapped in GoldenConfigEntry

2. `get_backup_config(client, device_name_or_id: str) -> GoldenConfigEntry`
   - Same as above but caller focuses on backup_config field
   - Same API call, separate function for clarity

**Compliance Features CRUD (GC-03):**

3. `list_compliance_features(client) -> ListResponse[ComplianceFeatureSummary]`
   - `features = client.golden_config.compliance_feature.all()`

4. `create_compliance_feature(client, name: str, slug: str, description: str = "") -> ComplianceFeatureSummary`
   - `feature = client.golden_config.compliance_feature.create(name=name, slug=slug, description=description)`

5. `delete_compliance_feature(client, feature_id: str) -> dict`
   - `client.golden_config.compliance_feature.get(id=feature_id).delete()`
   - Return `{"success": True, "message": f"Compliance feature {feature_id} deleted"}`

**Compliance Rules CRUD (GC-04):**

6. `list_compliance_rules(client, feature: str | None = None, platform: str | None = None) -> ListResponse[ComplianceRuleSummary]`
   - Filter by feature and/or platform if provided
   - `rules = client.golden_config.compliance_rule.filter(**filters)`

7. `create_compliance_rule(client, feature: str, platform: str, config_ordered: bool = False, match_config: str = "", description: str = "") -> ComplianceRuleSummary`
   - `rule = client.golden_config.compliance_rule.create(...)`

8. `update_compliance_rule(client, rule_id: str, **updates) -> ComplianceRuleSummary`
   - `rule = client.golden_config.compliance_rule.get(id=rule_id)`
   - `rule.update(updates)`

9. `delete_compliance_rule(client, rule_id: str) -> dict`
   - Return `{"success": True, "message": f"Compliance rule {rule_id} deleted"}`

**Compliance Checks (GC-05, GC-06):**

10. `get_compliance_results(client, device_name_or_id: str) -> ComplianceResult`
    - `results = client.golden_config.config_compliance.filter(device=device_name_or_id)`
    - Return `ComplianceResult.from_nautobot(device_name, results)`

11. `quick_diff_config(client, device_name_or_id: str) -> ComplianceResult`
    - Pull intended + backup configs for device
    - Use `difflib.unified_diff()` to compare sections
    - Build `ComplianceResult` with `source="quick-diff"` and `ConfigDiff` objects in a simplified format

All functions use try/except wrapping with `client._handle_api_error(e, action, resource_type)`.
