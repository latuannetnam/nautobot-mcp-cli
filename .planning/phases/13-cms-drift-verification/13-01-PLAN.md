# Plan 13-01: CMS Drift Engine — DiffSync Models, Adapters & CMSDriftReport

<objective>
Create the CMS drift comparison engine using DiffSync adapters to compare
live Juniper device state (agent-provided structured data) against Nautobot
CMS model records. Covers both BGP neighbor drift (DRIFT-01) and static
route drift (DRIFT-02).
</objective>

<must_haves>
- DiffSync models for BGP neighbor comparison (identity: peer_ip; attributes: peer_as, local_address, group_name)
- DiffSync models for static route comparison (identity: destination; attributes: nexthops, preference, metric, routing_instance)
- LiveBGPAdapter and CMSBGPAdapter DiffSync adapters
- LiveStaticRouteAdapter and CMSStaticRouteAdapter DiffSync adapters
- CMSDriftReport Pydantic model with bgp_neighbors and static_routes DriftSection fields
- compare_bgp_neighbors() function accepting live data dict/list + device name
- compare_static_routes() function accepting live data dict/list + device name
- Input normalization for jmcp output formats
</must_haves>

<action>
In `nautobot_mcp/cms/cms_drift.py`, create DiffSync models:

```python
from diffsync import Adapter, DiffSyncModel

class SyncBGPNeighbor(DiffSyncModel):
    """DiffSync model for BGP neighbor comparison."""
    _modelname = "bgp_neighbor"
    _identifiers = ("peer_ip",)
    _attributes = ("peer_as", "local_address", "group_name")

    peer_ip: str
    peer_as: int = 0
    local_address: str = ""
    group_name: str = ""

class SyncStaticRoute(DiffSyncModel):
    """DiffSync model for static route comparison."""
    _modelname = "static_route"
    _identifiers = ("destination",)
    _attributes = ("nexthops_str", "preference", "metric", "routing_instance")

    destination: str
    nexthops_str: str = ""  # Serialized sorted nexthop list for comparison
    preference: int = 5
    metric: int = 0
    routing_instance: str = ""
```

Note: `nexthops_str` is a serialized string of sorted nexthop IPs because
DiffSync attributes must be hashable. The comparison function will serialize
the nexthop list to a consistent string for both sides.
</action>

### Task 3: Create DiffSync adapters for live and CMS data

<read_first>
- nautobot_mcp/verification.py (ParsedConfigAdapter, NautobotLiveAdapter — adapter pattern with load())

</read_first>

<action>
In `nautobot_mcp/cms/cms_drift.py`, create 4 adapter classes:

**LiveBGPAdapter(Adapter):**
- `top_level = ["bgp_neighbor"]`
- `bgp_neighbor = SyncBGPNeighbor`
- Accepts `live_data: list[dict]` — pre-fetched jmcp output
- `load()`: Iterate live_data, extract peer_ip, peer_as, local_address, group_name and add SyncBGPNeighbor instances.
- Normalize peer_ip (strip trailing whitespace, etc.)

**CMSBGPAdapter(Adapter):**
- `top_level = ["bgp_neighbor"]`
- `bgp_neighbor = SyncBGPNeighbor`
- Accepts `client: NautobotClient` and `device_name: str`
- `load()`: Call `list_bgp_neighbors(client, device=device_name)` then `list_bgp_groups(client, device=device_name)` to build a group_id→name map. Add SyncBGPNeighbor for each CMS record.

**LiveStaticRouteAdapter(Adapter):**
- `top_level = ["static_route"]`
- `static_route = SyncStaticRoute`
- Accepts `live_data: list[dict]`
- `load()`: Iterate live_data, extract destination, nexthops (serialize to sorted comma-joined string), preference, metric, routing_instance. Add SyncStaticRoute instances.

**CMSStaticRouteAdapter(Adapter):**
- `top_level = ["static_route"]`
- `static_route = SyncStaticRoute`
- Accepts `client: NautobotClient` and `device_name: str`
- `load()`: Call `list_static_routes(client, device=device_name)`. Serialize nexthop IPs from inlined nexthops to matching sorted string format. Add SyncStaticRoute instances.

Helper: `_serialize_nexthops(nexthops: list) -> str` — sorts nexthop IPs alphabetically and joins with comma.
</action>

```python
def compare_bgp_neighbors(
    client: NautobotClient,
    device_name: str,
    live_neighbors: list[dict],
) -> CMSDriftReport:
    """Compare live BGP neighbors against Nautobot CMS records.

    Args:
        client: NautobotClient instance.
        device_name: Device hostname in Nautobot.
        live_neighbors: List of dicts from jmcp, each with keys:
            peer_ip (str), peer_as (int), local_address (str), group_name (str).

    Returns:
        CMSDriftReport with bgp_neighbors section populated.
    """
```

Implementation:
1. Create `LiveBGPAdapter`, set `live_data=live_neighbors`, call `load()`
2. Create `CMSBGPAdapter`, set `client=client`, `device_name=device_name`, call `load()`
3. `diff = cms_adapter.diff_from(live_adapter)`
4. Translate diff to `CMSDriftReport` using a `_diffsync_to_cms_drift()` helper
5. Build summary, return report

```python
def compare_static_routes(
    client: NautobotClient,
    device_name: str,
    live_routes: list[dict],
) -> CMSDriftReport:
    """Compare live static routes against Nautobot CMS records.

    Args:
        client: NautobotClient instance.
        device_name: Device hostname in Nautobot.
        live_routes: List of dicts from jmcp, each with keys:
            destination (str), nexthops (list[str]), preference (int),
            metric (int), routing_instance (str).

    Returns:
        CMSDriftReport with static_routes section populated.
    """
```

Same pattern as BGP but with static route adapters.

Create helper `_diffsync_to_cms_drift(diff, device_name, section_name)` that translates

in `verification.py` but targeting `bgp_neighbors` or `static_routes` section.

Create helper `_build_cms_summary(report: CMSDriftReport) -> dict` that counts
total drifts across bgp_neighbors and static_routes sections.
</action>
