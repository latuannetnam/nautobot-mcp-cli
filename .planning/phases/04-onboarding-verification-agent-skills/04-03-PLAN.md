Create `.agent/skills/onboard-router-config/SKILL.md` — agent-side orchestration guide.

```markdown
---
name: onboard-router-config
description: Parse live router config from jmcp and onboard into Nautobot
---

## Prerequisites
- nautobot-mcp-cli MCP server configured and running
- jmcp MCP server configured and running (for live config pull)
- Target router accessible via jmcp

## Workflow Steps

### Step 1: Get device list from Nautobot
Call `nautobot_list_devices()` to verify the target device or identify devices to onboard.

### Step 2: Pull live config from router via jmcp
Call `execute_junos_command(router_name="<device>", command="show configuration | display json")`.
Save the JSON output.

### Step 3: Parse the config
Call `nautobot_parse_config(config_json=<jmcp_output>, network_os="juniper_junos")`.
Review the parsed result: interfaces, IPs, VLANs, system info.

### Step 4: Dry-run onboarding
Call `nautobot_onboard_config(device_name="<device>", parsed_config_json=<parsed_output>, dry_run=true)`.
Review the plan:
- **Summary**: "Will create: N interfaces, N IPs. Will update: N. Will skip: N."
- **Actions**: Detailed table of each planned change.
Present the dry-run results to the user for approval.

### Step 5: Commit (if approved)
Call `nautobot_onboard_config(device_name="<device>", parsed_config_json=<parsed_output>, dry_run=false)`.
Report results: created/updated/skipped/failed counts.

## Progress Reporting
After each step, report:
- ✅ Step N complete: [brief result summary]
- ⏭️ Next: Step N+1

## Error Handling
- **Device not found in jmcp**: Verify router_name matches jmcp device list
- **Parse errors**: Check if config is valid JSON from `show configuration | display json`
- **Nautobot conflicts**: If device type or location doesn't exist, offer to create
- **Partial failure**: Report which objects failed and continue with remaining
```

Create `.agent/skills/verify-compliance/SKILL.md` — agent-side orchestration guide.

```markdown
---
name: verify-compliance
description: Compare live router config against Nautobot Golden Config and data model
---

## Prerequisites
- nautobot-mcp-cli MCP server configured and running
- jmcp MCP server configured and running (for live config pull)
- Golden Config plugin configured with intended configs

## Workflow Steps

### Step 1: Check Golden Config compliance
Call `nautobot_verify_config_compliance(device_name="<device>")`.
Review: compliant/non-compliant status, diff details.

### Step 2: Pull live config from router
Call `execute_junos_command(router_name="<device>", command="show configuration | display json")`.

### Step 3: Parse the config
Call `nautobot_parse_config(config_json=<jmcp_output>, network_os="juniper_junos")`.

### Step 4: Verify data model consistency
Call `nautobot_verify_data_model(device_name="<device>", parsed_config_json=<parsed_output>)`.
Review drift report:
- **Interfaces**: missing, extra, changed
- **IP Addresses**: missing, extra, changed
- **VLANs**: missing, extra, changed

### Step 5: Present combined report
Combine results from Steps 1 and 4:
- Config compliance status (intended vs actual)
- Data model drift (live vs Nautobot records)
- Actionable recommendations

## Progress Reporting
After each step, report:
- ✅ Step N complete: [brief result summary]
- ⏭️ Next: Step N+1

## Error Handling
- **No Golden Config entry**: Device may not have intended config generated yet
- **jmcp unreachable**: User can provide config file manually instead
- **Large diff**: Summarize top-level changes, link full report for details
```
