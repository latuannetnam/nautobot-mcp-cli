# Plan 04-01: Config Onboarding Engine

<objective>
Create the config onboarding engine that takes a ParsedConfig (from Phase 3 parser) and creates/updates Nautobot objects. Supports dry-run (default) and commit modes, idempotent matching by name+device, and auto-resolution of prerequisites (device types, prefixes, etc.).
</objective>

<must_haves>
- Pydantic models for onboarding actions and results
- Onboarding engine with dry-run (default) and commit modes
- Idempotent matching: interfaces by (device_name, interface_name), IPs by (address, device)
- Auto-create smallest containing prefix for IPs
- Auto-detect and resolve device type, manufacturer, location from parsed config + Nautobot lookup
- `--update-existing` flag support (default: skip conflicts)
- Summary + detail action table output
</must_haves>

Add an interface type mapping function to `nautobot_mcp/onboarding.py` (or a helper module). JunOS interface names need to map to Nautobot interface types.

```python
JUNOS_INTERFACE_TYPE_MAP = {
    "ge-": "1000base-t",
    "xe-": "10gbase-x-sfpp",
    "et-": "25gbase-x-sfp28",
    "ae": "lag",
    "lo": "virtual",
    "irb": "virtual",
    "vlan": "virtual",
    "me": "1000base-t",
    "fxp": "1000base-t",
    "em": "1000base-t",
}

def map_interface_type(interface_name: str) -> str:
    """Map JunOS interface name prefix to Nautobot interface type."""
    for prefix, iface_type in JUNOS_INTERFACE_TYPE_MAP.items():
        if interface_name.startswith(prefix):
            return iface_type
    return "other"
```

<action>
Create `nautobot_mcp/onboarding.py` with the main onboarding engine. The core function is `onboard_config()`:

```python
def onboard_config(
    client: NautobotClient,
    parsed_config: ParsedConfig,
    device_name: str,
    dry_run: bool = True,
    update_existing: bool = False,
    location: str | None = None,
    device_type: str | None = None,
    role: str = "Router",
    namespace: str = "Global",
) -> OnboardResult:
```

**Internal helper functions:**

1. `_resolve_device(client, device_name, parsed_config, location, device_type, role, dry_run) -> tuple[OnboardAction, str | None]`
   - Try `get_device(client, name=device_name)` to find existing
   - If not found: plan create with auto-detected device_type from `parsed_config.platform`
   - If found: return skip action with existing device ID
   - Use `parsed_config.system.hostname` as fallback for device_name

2. `_resolve_interfaces(client, device_id, parsed_config, dry_run, update_existing) -> list[OnboardAction]`
   - For each `ParsedInterface` in parsed_config:
     - Check if interface exists: `client.api.dcim.interfaces.get(device_id=device_id, name=iface.name)`
     - If not found: plan create with `map_interface_type(iface.name)`
     - If found + different attributes + update_existing: plan update
     - If found + same attributes: plan skip

3. `_resolve_ip_addresses(client, device_id, parsed_config, namespace, dry_run) -> list[OnboardAction]`
   - For each interface unit's IP addresses:
     - Auto-create prefix using `ipaddress.ip_interface(addr).network`
     - Check if IP exists: `client.api.ipam.ip_addresses.get(address=addr)`
     - If not found: plan create + assign to interface
     - If found: plan skip

4. `_resolve_vlans(client, parsed_config, dry_run) -> list[OnboardAction]`
   - For each `ParsedVLAN`:
     - Check if VLAN exists: `client.api.ipam.vlans.get(vid=vlan.vlan_id)`
     - If not found: plan create
     - If found: plan skip

**Execution logic:**
- If `dry_run=True`: collect all actions, build summary, return result
- If `dry_run=False`: execute each non-skip action using existing CRUD functions, catch errors per-action
</action>
