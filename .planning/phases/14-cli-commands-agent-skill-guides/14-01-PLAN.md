# Plan 14-01: Extract Drift CLI to Dedicated Module & Add ARP CLI

<objective>
Move drift comparison CLI commands (`drift-bgp`, `drift-routes`) from `cms_routing.py` to a new dedicated `cms_drift.py` CLI module registered at `nautobot-mcp cms drift`. Add ARP CLI subgroup (`nautobot-mcp cms arp`). This cleans up separation of concerns per user decision 1B and ensures all CMS domains have CLI coverage (CLI-01).
</objective>

## Tasks

<task id="14-01-01">
<title>Create cms_drift.py CLI module with drift commands</title>
<read_first>
- nautobot_mcp/cli/cms_routing.py (lines 442-548 — drift commands to extract)
- nautobot_mcp/cli/app.py (registration pattern, lines 144-150)
- nautobot_mcp/cms/cms_drift.py (drift engine functions: compare_bgp_neighbors, compare_static_routes)
</read_first>
<action>
Create `nautobot_mcp/cli/cms_drift.py` with:

1. Import structure matching existing CMS CLI pattern:
   ```python
   from nautobot_mcp.cli.app import get_client_from_ctx, handle_cli_error
   from nautobot_mcp.cms.cms_drift import compare_bgp_neighbors, compare_static_routes
   ```

2. `drift_app = typer.Typer(help="CMS drift verification commands")`

3. Move `drift-bgp` command (currently `drift_bgp_cmd` at cms_routing.py L447-495):
   - Command name: `bgp`
   - Args: `--device` (required), `--from-file` (optional, falls back to stdin)
   - JSON and table output modes via `ctx.obj.get("json", False)`
   - Table output showing: device name, total drifts, missing/extra/changed counts
   - Detail sections for missing, extra, changed items

4. Move `drift-routes` command (currently `drift_routes_cmd` at cms_routing.py L498-548):
   - Command name: `routes`
   - Args: `--device` (required), `--from-file` (optional, falls back to stdin)
   - Same output format as bgp drift command

Both commands read JSON from `--from-file` path or `sys.stdin.read()` as fallback.
</action>
<acceptance_criteria>
- `nautobot_mcp/cli/cms_drift.py` exists and contains `drift_app = typer.Typer(`
- File contains `@drift_app.command("bgp")` and `@drift_app.command("routes")`
- File imports `compare_bgp_neighbors` and `compare_static_routes` from `nautobot_mcp.cms.cms_drift`
- File imports `get_client_from_ctx` and `handle_cli_error` from `nautobot_mcp.cli.app`
- Both commands accept `--device` and `--from-file` options
- Both commands handle stdin fallback via `sys.stdin.read()`
</acceptance_criteria>
</task>
