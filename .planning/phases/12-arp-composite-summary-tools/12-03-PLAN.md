## Task 1: Implement `get_interface_detail()` in interfaces.py

<read_first>
- nautobot_mcp/cms/interfaces.py (existing list_interface_units, list_interface_families, list_interface_family_filters, list_interface_family_policers, list_vrrp_groups functions)
- nautobot_mcp/cms/arp.py (list_arp_entries — for ARP data on the interface)
- nautobot_mcp/models/cms/composites.py (InterfaceDetailResponse model)
- nautobot_mcp/devices.py § get_device_summary (composite pattern reference)
</read_first>

<action>
Add `get_interface_detail()` function to `nautobot_mcp/cms/interfaces.py`:

```python
def get_interface_detail(
    client: NautobotClient,
    unit_id: Optional[str] = None,
    device: Optional[str] = None,
    interface_name: Optional[str] = None,
    detail: bool = False,
) -> InterfaceDetailResponse:
```

**Implementation:**
1. If `unit_id` provided: call `get_interface_unit(client, id=unit_id)` to get unit
2. If `device` + `interface_name` provided: call `list_interface_units(client, device=device)` and find matching unit by name
3. Get unit data (id, name, description, mtu, etc.)
4. Call `list_interface_families(client, unit=unit.id)` → families
5. Call `list_vrrp_groups(client, unit=unit.id)` → vrrp_groups
6. Call `list_arp_entries(client, interface=unit.interface_id)` → arp_entries (from cms.arp module)
7. **Default mode (detail=False):**
   - Unit info as dict
   - Family list with names only
   - Counts for filters, policers, VRRP groups, ARP entries
8. **Detail mode (detail=True):**
   - For each family: call `list_interface_family_filters(client, family=family.id)` and `list_interface_family_policers(client, family=family.id)`
   - Inline full filter names, policer settings, VRRP virtual IPs, ARP entries
9. Return `InterfaceDetailResponse(unit=..., families=..., vrrp_groups=..., arp_entries=..., family_count=..., vrrp_count=..., arp_count=...)`

Import `InterfaceDetailResponse` from `nautobot_mcp.models.cms.composites`.
Import `list_arp_entries` from `nautobot_mcp.cms.arp`.
</action>

<acceptance_criteria>
- `interfaces.py` contains `def get_interface_detail(`
- `interfaces.py` contains `detail: bool = False`
- `interfaces.py` contains `InterfaceDetailResponse`
- `interfaces.py` contains `from nautobot_mcp.cms.arp import list_arp_entries` or equivalent import
- `python -c "from nautobot_mcp.cms.interfaces import get_interface_detail"` exits 0
</acceptance_criteria>

## Task 2: Implement `get_device_firewall_summary()` in firewalls.py

<read_first>
- nautobot_mcp/cms/firewalls.py (existing list_firewall_filters, list_firewall_terms, list_firewall_policers functions)
- nautobot_mcp/models/cms/composites.py (FirewallSummaryResponse model)
</read_first>

<action>
Add `get_device_firewall_summary()` function to `nautobot_mcp/cms/firewalls.py`:

```python
def get_device_firewall_summary(
    client: NautobotClient,
    device: str,
    detail: bool = False,
) -> FirewallSummaryResponse:
```

**Implementation:**
1. Call `resolve_device_id(client, device)` to get device UUID
2. Call `list_firewall_filters(client, device=device)` → filters
3. Call `list_firewall_policers(client, device=device)` → policers
4. **Default mode (detail=False):**
   - Filters include: name, family, term_count (from list which already has count)
   - Policers include: name, bandwidth_limit, bandwidth_unit, burst_limit, burst_unit
5. **Detail mode (detail=True):**
   - For each filter: call `list_firewall_terms(client, filter=filter.id)` and inline term names with match/action summary
   - Include term-level details: name, order, enabled, match_count, action_count
6. Return `FirewallSummaryResponse(device_name=device, filters=..., policers=..., total_filters=..., total_policers=...)`

Import `FirewallSummaryResponse` from `nautobot_mcp.models.cms.composites`.
</action>

<acceptance_criteria>

- `firewalls.py` contains `detail: bool = False`
- `firewalls.py` contains `FirewallSummaryResponse`
