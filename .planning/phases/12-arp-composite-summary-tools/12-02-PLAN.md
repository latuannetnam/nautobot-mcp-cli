## Task 1: Implement `get_device_bgp_summary()` in routing.py

<read_first>
- nautobot_mcp/cms/routing.py (existing list_bgp_groups, list_bgp_neighbors, list_bgp_address_families, list_bgp_policy_associations functions)
- nautobot_mcp/models/cms/composites.py (BGPSummaryResponse model)
- nautobot_mcp/devices.py § get_device_summary (existing composite pattern: compose multiple API calls into one response)
</read_first>

<action>
Add `get_device_bgp_summary()` function to `nautobot_mcp/cms/routing.py`:

```python
def get_device_bgp_summary(
    client: NautobotClient,
    device: str,
    detail: bool = False,
) -> BGPSummaryResponse:
```

**Implementation:**
1. Call `resolve_device_id(client, device)` to get device UUID
2. Call `list_bgp_groups(client, device=device)` → groups
3. Call `list_bgp_neighbors(client, device=device)` → neighbors
4. For each group: attach matching neighbors (filtered by group id)
5. **Default mode (detail=False):**
   - Groups include: name, type, local_as, neighbor count
   - Neighbors include: peer_address, peer_as, group_name (session state if available)
   - Address family and policy counts only
6. **Detail mode (detail=True):**
   - For each neighbor: call `list_bgp_address_families(client, neighbor=neighbor.id)` and `list_bgp_policy_associations(client, neighbor=neighbor.id)`
   - Inline full address family and policy association data per neighbor
7. Return `BGPSummaryResponse(device_name=device, groups=..., total_groups=..., total_neighbors=...)`

Import `BGPSummaryResponse` from `nautobot_mcp.models.cms.composites`.
</action>

<acceptance_criteria>
- `routing.py` contains `def get_device_bgp_summary(`
- `routing.py` contains `detail: bool = False`
- `routing.py` contains `BGPSummaryResponse`
- `python -c "from nautobot_mcp.cms.routing import get_device_bgp_summary"` exits 0
</acceptance_criteria>

- nautobot_mcp/models/cms/composites.py (RoutingTableResponse model)
</read_first>

<action>
Add `get_device_routing_table()` function to `nautobot_mcp/cms/routing.py`:

```python
def get_device_routing_table(
    client: NautobotClient,
    device: str,
    detail: bool = False,
) -> RoutingTableResponse:
```

**Implementation:**
1. Call `resolve_device_id(client, device)` to get device UUID
2. Call `list_static_routes(client, device=device)` → routes
3. **Default mode (detail=False):**
   - Routes include: destination, preference, routing_instance_name, nexthop_count (count only)
4. **Detail mode (detail=True):**
   - For each route: call `list_static_route_nexthops(client, route=route.id)` and `list_static_route_qualified_nexthops(client, route=route.id)`
   - Inline all simple and qualified next-hops per route
5. Return `RoutingTableResponse(device_name=device, routes=..., total_routes=...)`

Import `RoutingTableResponse` from `nautobot_mcp.models.cms.composites`.
</action>
