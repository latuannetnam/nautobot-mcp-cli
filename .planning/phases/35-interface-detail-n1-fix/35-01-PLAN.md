## Context

**Problem:** `get_interface_detail()` (L689-692) calls `list_interface_families(client, unit_id=unit.id)` **per unit** in a loop. With ~2,000 units on HQV-PE1: ~2,000 HTTP calls.

**Fix:** After the `units_resp` fetch (L684), before the unit loop, prefetch ALL families for the device in a single bulk call. Build a `unit_id → InterfaceFamilySummary[]` lookup map. Replace the per-unit loop with map lookups.

**Contract goal (CQP-01):** `get_interface_detail` makes ≤3 HTTP calls total (1 units + 1 families + 1 VRRP bulk, regardless of unit count).

## Action

In `nautobot_mcp/cms/interfaces.py`, in `get_interface_detail()`, after line 685 (`units = units_resp.results`):

### Step 1 — Resolve `device_id` (needed for bulk family prefetch)

`units_resp` does NOT expose `device_id` — `list_interface_units` resolves it internally. Call `resolve_device_id` once:

```python
# L686 — after: units = units_resp.results
device_id = resolve_device_id(client, device)
```

### Step 2 — Bulk fetch all families for the device (replaces L689-692 per-unit loop)

```python
# L687-691 — replace the per-unit loop (L689-692):
#   for unit in units:
#       families = list_interface_families(client, unit_id=unit.id, limit=0)
#       unit_families[unit.id] = families.results

all_families_resp = cms_list(
    client,
    "juniper_interface_families",
    InterfaceFamilySummary,
    device=device_id,
    limit=0,
)

# Index all families by unit_id (same pattern as list_interface_units L68-80)
unit_families: dict[str, list[InterfaceFamilySummary]] = {}
for fam in all_families_resp.results:
    unit_families.setdefault(fam.unit_id, []).append(fam)
```

### Step 3 — Remove the per-unit family loop (L689-692)

Delete these lines:
```python
unit_families: dict[str, list] = {}
for unit in units:
    families = list_interface_families(client, unit_id=unit.id, limit=0)
    unit_families[unit.id] = families.results
```

The `unit_families` dict is now built by the bulk prefetch. The rest of the function (L694+ `_get_vrrp_for_family`, L709-738 unit processing loop) is unchanged — it already uses `unit_families.get(unit.id, [])`.

### Step 4 — Update the `vrrp_by_family` declaration (L694)

Move `vrrp_by_family` declaration to immediately before the prefetch block so it is in scope when `_get_vrrp_for_family` is defined in Plan 02. Change:

```python
# OLD L694:
vrrp_by_family: dict[str, list] = {}
```

to:

```python
# NEW L687 (inserted just before the all_families_resp block):
vrrp_by_family: dict[str, list[VRRPGroupSummary]] = {}
```

This declaration is currently at L694, after the family loop. Moving it up makes it available to both the family prefetch block (Plan 01) and the VRRP prefetch block (Plan 02). The variable is still assigned in Plan 02's VRRP prefetch block.
