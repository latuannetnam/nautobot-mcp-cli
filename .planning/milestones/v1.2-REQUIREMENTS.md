# Requirements: nautobot-mcp-cli v1.2

**Defined:** 2026-03-20
**Core Value:** AI agents can read and write Nautobot data through standardized MCP tools — including Juniper-specific CMS models

## v1.2 Requirements

Requirements for Juniper CMS Model MCP Tools release. Each maps to roadmap phases.

### Routing

- [x] **RTG-01**: Agent can list and filter Juniper static routes for a device
- [x] **RTG-02**: Agent can create, update, and delete Juniper static routes with next-hops
- [x] **RTG-03**: Agent can list and filter BGP groups for a device (with type, local-address, cluster-id)
- [x] **RTG-04**: Agent can create, update, and delete BGP groups
- [x] **RTG-05**: Agent can list and filter BGP neighbors for a device (with peer IP, AS, session state)
- [x] **RTG-06**: Agent can create, update, and delete BGP neighbors
- [x] **RTG-07**: Agent can list BGP address families for a group
- [x] **RTG-08**: Agent can list BGP policy associations for a group or neighbor
- [x] **RTG-09**: Agent can list BGP received routes for a neighbor
- [x] **RTG-10**: Agent can list and manage routing instances for a device (VRF, VPLS, virtual-router)

### Interfaces

- [x] **INTF-01**: Agent can list and filter Juniper interface units for a device (VLAN mode, encapsulation, QinQ)
- [x] **INTF-02**: Agent can create, update, and delete interface units with VLAN configuration
- [x] **INTF-03**: Agent can list interface families for an interface unit (inet, inet6, mpls)
- [x] **INTF-04**: Agent can manage interface family filter associations (input/output filters)
- [x] **INTF-05**: Agent can manage interface family policer associations (input/output/arp policers)
- [x] **INTF-06**: Agent can list and manage VRRP groups for an interface (virtual IP, priority, tracking)
- [x] **INTF-07**: Agent can list and manage VRRP track routes and track interfaces

### Firewalls

- [x] **FW-01**: Agent can list and filter firewall filters for a device (with family, term count)
- [x] **FW-02**: Agent can create, update, and delete firewall filters
- [x] **FW-03**: Agent can list and manage firewall terms within a filter (ordered, with match/action counts)
- [x] **FW-04**: Agent can list and manage match conditions within a term (source-address, protocol, port, etc.)
- [x] **FW-05**: Agent can list and manage actions within a term (accept, discard, reject, count, policer)
- [x] **FW-06**: Agent can list and manage firewall policers for a device (bandwidth, burst, actions)
- [x] **FW-07**: Agent can list match condition to prefix list associations

### Policies

- [x] **POL-01**: Agent can list and filter policy statements for a device
- [x] **POL-02**: Agent can create, update, and delete policy statements
- [x] **POL-03**: Agent can list and manage JPS terms within a policy statement (ordered, with conditions/actions)
- [x] **POL-04**: Agent can list and manage JPS match conditions (route-filter, prefix-list, community, as-path)
- [x] **POL-05**: Agent can list and manage JPS actions (accept, reject, local-preference, community, as-path)
- [x] **POL-06**: Agent can list and manage policy prefix lists and prefix associations for a device
- [x] **POL-07**: Agent can list and manage policy communities for a device
- [x] **POL-08**: Agent can list and manage policy AS paths for a device

### ARP

- [x] **ARP-01**: Agent can list and filter ARP entries for a device or interface (MAC, IP, hostname)
- [x] **ARP-02**: Agent can create, update, and delete ARP entries

### Composite Tools

- [x] **COMP-01**: Agent can get a full BGP summary for a device in one call (groups → neighbors → session state → prefix counts)
- [x] **COMP-02**: Agent can get a full routing table summary for a device (static routes with all next-hops, routing instances)
- [x] **COMP-03**: Agent can get full interface detail in one call (unit → families → filters → policers → VRRP → ARP)
- [x] **COMP-04**: Agent can get a firewall summary for a device (filters → terms count, policers, interface associations)

### Drift Verification

- [x] **DRIFT-01**: Agent can compare live BGP neighbors (from jmcp) against Nautobot CMS BGP model records
- [x] **DRIFT-02**: Agent can compare live static routes (from jmcp) against Nautobot CMS static route records

### CLI & Skills

- [x] **CLI-01**: CLI commands expose all CMS model tools (routing, interfaces, firewalls, policies, ARP)
- [x] **CLI-02**: CLI commands expose composite summary tools
- [x] **CLI-03**: CLI commands expose CMS drift verification
- [x] **SKILL-01**: Agent skill guide for CMS-aware device audit workflow

## v2 Requirements

Deferred to future release. Tracked but not in current roadmap.

### Multi-Vendor

- **MV-01**: Agent can parse and onboard Cisco IOS configurations
- **MV-02**: Agent can parse and onboard Arista EOS configurations

### Bulk Operations

- **BULK-01**: Agent can onboard multiple device configs in a batch
- **BULK-02**: Agent can run drift comparison across a fleet of devices

### Remediation

- **REM-01**: Agent can suggest config remediation based on drift reports

## Out of Scope

| Feature | Reason |
|---------|--------|
| Direct device communication | Handled by jmcp; this tool consumes Nautobot APIs only |
| Modifying netnam-cms-core plugin | We consume the REST API, not modify the Django plugin |
| Nautobot server management | Server already deployed and managed separately |
| Auto-remediation | Reports drift only; human approval required for changes |
| Non-Juniper CMS models | v1.2 focuses on Juniper; other vendors are v2+ |
