## Device-Scoped IP Query (DEVIP)

- [x] **DEVIP-01**: `nautobot_get_device_ips(device_name)` returns all IP addresses assigned to a device's
  interfaces in one MCP call, returning structured list of {interface, address, status}
  → _Implemented via M2M traversal; `ip_addresses.filter(device=...)` was unreliable_
- [x] **DEVIP-02**: Result is JSON-serializable so agents can pass it directly to drift comparison without
  marshalling
  → _Returns `DeviceIPsResponse.model_dump()` which is fully serializable_
- [x] **DEVIP-03**: CLI `nautobot-mcp ipam ips list --device DEVICE` surfaces the same data
  → _Added `--device` option to `ipam ips` command_

## Filtering Cross-Entity (FILT)

- [x] **FILT-01**: `list_ip_addresses(device=...)` filters IPs by device via interface traversal
  → _Added device param to `list_ip_addresses` using M2M path_
- [x] **FILT-02**: `list_vlans(device=...)` scopes VLAN list to a specific device's interfaces
  → _Walks interface `untagged_vlan` + `tagged_vlans` fields_
- [x] **FILT-03**: `list_interfaces(device=...)` already existed; verified stability after enrichment changes
  → _No regression in existing behavior_

## Device Summary (SUMM)

- [x] **SUMM-01**: `nautobot_get_device_summary(device_name)` returns interface count, IP count, VLAN count,
  and link state statistics (up/down)
  → _Implemented; returns `DeviceSummary` Pydantic model_
- [x] **SUMM-02**: Works as single-call answer to "tell me everything about this device"
  → _One call returns all counts and basic health_
- [x] **SUMM-03**: CLI `devices summary DEVICE` and `--detail` flag for full breakdown
  → _Added to `cli/devices.py`_

## Drift Comparison (DRIFT)

- [x] **DRIFT-01**: `nautobot_compare_device(device_name, interfaces_data)` accepts dict (interface→IPs/VLANs)
  and returns structured diff
  → _Implemented with flat map auto-detection_
- [x] **DRIFT-02**: Drift tool returns per-interface `{missing_ips, extra_ips, missing_vlans, extra_vlans}`
  plus global summary `{total_drifts, interfaces_checked, interfaces_with_drift}`
  → _`InterfaceDrift` + `DriftSummary` + `QuickDriftReport` Pydantic models_
- [x] **DRIFT-03**: Works with arbitrary input data — accepts DeviceIPEntry list directly from
  `nautobot_get_device_ips` with no transformation required
  → _`isinstance(data, list)` auto-detects shape_
- [x] **DRIFT-04**: `nautobot-mcp verify quick-drift DEVICE --interface X --ip Y` CLI command works;
  agent can chain jmcp → parse → MCP drift check — zero Python scripting required
  → _CLI with --interface/--ip/--vlan/--data/--file/stdin modes_

## Enriched Interface Data (ENRICH)

- [x] **ENRICH-01**: `nautobot_list_interfaces` with `include_ips=True` returns each interface with its
  assigned IPs embedded
  → _Batch M2M query pattern; `ip_addresses` field populated when flag is set_
- [x] **ENRICH-02**: Interface IP enrichment uses efficient batch query (not N+1 per interface)
  → _Single `ip_address_to_interface.filter(interface_id__in=[...])` call for all interfaces_
